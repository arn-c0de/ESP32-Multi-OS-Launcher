# SD Card Firmware Packaging

To use the Multi-OS Launcher, your SD card must be prepared correctly with the operating systems you want to flash.

## 1. Folder Structure

By default, the launcher expects all firmware files to be in a folder named `/firmware` in the root of the SD card.

```text
SD_ROOT/
└── firmware/
    ├── app_a.bin
    ├── app_b.bin
    └── linux_esp32.bin
```

*Note: You can change this directory name (e.g., to `/os` or `/Firmwares`) using the Kconfig settings in your ESP-IDF project.*

## 2. Firmware Format

The firmware must be a standard ESP32 binary (`.bin`) generated by the ESP-IDF or Arduino IDE.

### For ESP-IDF:
Use the binary located in your project's `build/` folder (e.g., `build/my_app.bin`).

### For Arduino IDE:
1. Go to **Sketch** -> **Export compiled Binary**.
2. Find the generated `.bin` file in your project folder.

## 3. Filenames

- Files must end with the extension **`.bin`** (case-insensitive).
- Keep filenames descriptive but simple (max 64 characters recommended).
- The launcher will display these names in the selection menu.

## 4. Multi-OS Best Practices

To ensure every OS can flash back to others:
1. **Include the Launcher**: Make sure every OS you compile includes the `esp_launcher` component and calls `esp_launcher_check_and_run()` in its startup.
2. **Unified Partition Layout**: All operating systems should use the same or a compatible partition layout (OTA-0 and OTA-1).
3. **SD Card Pins**: Ensure all your projects use the same SPI pins for the SD card so the launcher can mount it regardless of which OS is running.


## Pushbuttons (AHH-1.0)

- Pushbutton 1: GPIO26
- Pushbutton 2: GPIO27
